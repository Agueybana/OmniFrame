# SWOT Skill

Use SWOT when the user needs a panoramic decision audit.

Root canvas expectations:
- Strengths must name assets, advantages, leverage, capabilities, relationships, resources, or patterns specific to the prompt.
- Weaknesses must name internal limitations, missing evidence, constraints, habits, execution gaps, or trust gaps specific to the prompt.
- Opportunities must name external openings, timing, partnerships, customer/user/user-life improvements, or environmental tailwinds.
- Threats must name external risks, substitutes, switching barriers, failure modes, social/legal/technical risks, or timing risks.

Sublayer expectations:
- Evidence: what would confirm or weaken the SWOT claim.
- Action: what the user should do next because of the claim.
- Metric: how the user will know the claim mattered.
