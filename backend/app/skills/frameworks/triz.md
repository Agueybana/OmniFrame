# TRIZ Skill

Use TRIZ when the request contains a contradiction, engineering paradox, design tradeoff, or tension between desirable properties.

Root canvas expectations:
- The contradiction must be written in the user's domain, not generic "improve target property."
- Selected principles must be relevant to the actual problem and translated into domain-specific moves.
- Include prototype tests and failure modes that prove the contradiction has been reduced.

Sublayer expectations:
- Contradiction options should state improve-X-without-worsening-Y.
- Inventive moves should translate a TRIZ principle into concrete structure, timing, mediation, parameter, material, process, or interface choices.
- Prototype and failure-mode panels must be specific to the user's context.
